# Module 3: POST — Creating a Resource

## Concept

POST is the HTTP method used to create a new resource on the server. Unlike GET, it requires sending data in the request body — the content you want the server to create.

## Test

Request: POST https://jsonplaceholder.typicode.com/posts
Body (raw JSON):
{
    "title": "My first API test",
    "body": "Learning how POST works",
    "userId": 1
}

Status: 201 Created

Response:
{
  "title": "My first API test",
  "body": "Learning how POST works",
  "userId": 1,
  "id": 101
}

## GET vs POST comparison

| | GET | POST |
|---|---|---|
| Purpose | Read existing data | Create new data |
| Body needed? | No | Yes |
| Typical success code | 200 OK | 201 Created |
| Response | The resource that already existed | The resource that now exists, with a new id |

## Key takeaway

> The client sends the content; the server assigns the identity. The id field in the response (101) was generated by the server, not something we specified — this is standard REST behavior for resource creation.
